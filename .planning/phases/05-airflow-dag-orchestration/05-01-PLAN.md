Create `scripts/airflow_setup.sh` — an idempotent bash script that installs Airflow 2.10.x on the EC2 instance and initializes the RDS PostgreSQL backend. The script must:

1. **Pre-flight checks:**
   - Verify swap is active: `free -h | grep Swap` — exit with error if swap total is 0 (INFRA-02 must be done first)
   - Verify Python 3.11 is available: `python3.11 --version`

2. **Install Airflow with constraints** (critical — no plain pip install):
   ```bash
   AIRFLOW_VERSION=2.10.4  # use latest 2.10.x available on PyPI
   PYTHON_VERSION=$(python3 --version | cut -d' ' -f2 | cut -d'.' -f1,2)
   CONSTRAINT_URL="https://raw.githubusercontent.com/apache/airflow/constraints-${AIRFLOW_VERSION}/constraints-${PYTHON_VERSION}.txt"
   pip install "apache-airflow[postgres]==${AIRFLOW_VERSION}" --constraint "${CONSTRAINT_URL}"
   ```

3. **Configure Airflow** — write `/opt/airflow/airflow.cfg` overrides via environment variables (not direct file edit, which is brittle):
   - `AIRFLOW_HOME=/opt/airflow`
   - `AIRFLOW__CORE__EXECUTOR=LocalExecutor`
   - `AIRFLOW__CORE__SQL_ALCHEMY_CONN=postgresql+psycopg2://airflow:${DB_PASS}@${RDS_ENDPOINT}:5432/airflow`
   - `AIRFLOW__WEBSERVER__WEB_SERVER_PORT=8080`
   - `AIRFLOW__SCHEDULER__CATCHUP_BY_DEFAULT=False`
   - `AIRFLOW__CORE__DAGS_FOLDER=/opt/airflow/dags`
   - `AIRFLOW__CORE__LOAD_EXAMPLES=False`  ← prevents example DAGs cluttering the UI

4. **Initialize RDS database:**
   - First verify RDS connectivity: `psql -h ${RDS_ENDPOINT} -U airflow -d postgres -c "\q"` — exit if fails
   - Create airflow database if not exists: `createdb -h ${RDS_ENDPOINT} -U airflow airflow` (idempotent: ignore "already exists")
   - Run: `airflow db migrate`
   - Create admin user (idempotent — skip if user already exists):
     ```bash
     airflow users create \
       --username admin --password "${AIRFLOW_ADMIN_PASSWORD}" \
       --firstname Admin --lastname User \
       --role Admin --email admin@example.com 2>/dev/null || true
     ```

5. **Copy DAG:**
   - `mkdir -p /opt/airflow/dags`
   - `cp dags/crypto_volatility_dag.py /opt/airflow/dags/`

6. **Register systemd services:**
   - Copy service files to `/etc/systemd/system/`
   - `systemctl daemon-reload`
   - `systemctl enable airflow-webserver airflow-scheduler`
   - `systemctl start airflow-webserver airflow-scheduler`

- `catchup=False`  (prevents backfill storm on restart)

**7 tasks in order** — each as `PythonOperator`:
1. `ingest` — calls `scripts/ingest.py`
2. `compute_features` — calls `scripts/compute_features.py`
3. `predict` — calls `scripts/predict.py`
4. `retrain` — calls `scripts/retrain.py`, **MUST have `trigger_rule=TriggerRule.NONE_FAILED_MIN_ONE_SUCCESS`**
5. `evaluate` — calls `scripts/evaluate.py`
6. `promote` — calls `scripts/promote.py`, **MUST have `trigger_rule=TriggerRule.NONE_FAILED_MIN_ONE_SUCCESS`**
7. `monitor` — calls `scripts/monitor.py`, **MUST have `trigger_rule=TriggerRule.ALL_DONE`** (always runs for observability)

**Each task callable pattern** — thin subprocess wrapper (scripts handle all business logic):
```python
import subprocess, sys, os

def run_script(script_name: str, **kwargs) -> None:
    """Generic callable for running a Phase 1-4 script as an Airflow task."""
    script_path = os.path.join(os.environ.get('PROJECT_ROOT', '/opt/airflow'), 'scripts', script_name)
    result = subprocess.run(
        [sys.executable, script_path],
        capture_output=True, text=True
    )
    print(result.stdout)
    if result.stderr:
        print(result.stderr)
    if result.returncode != 0:
        raise Exception(f"{script_name} failed with exit code {result.returncode}")
```

Use `functools.partial` or lambda to bind each script name:
```python
from functools import partial

ingest = PythonOperator(
    task_id='ingest',
    python_callable=partial(run_script, 'ingest.py'),
)
retrain = PythonOperator(
    task_id='retrain',
    python_callable=partial(run_script, 'retrain.py'),
    trigger_rule=TriggerRule.NONE_FAILED_MIN_ONE_SUCCESS,
)
```

**Full dependency chain** (must be exactly this):

```python
ingest >> compute_features >> predict >> retrain >> evaluate >> promote >> monitor
```
