```yaml
on:
  pull_request:
    branches: [main]

jobs:
  lint-and-test:
    name: Lint and Test
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Python 3.11
        uses: actions/setup-python@v5
        with:
          python-version: "3.11"
          cache: "pip"

      - name: Install dependencies
        run: |
          pip install -r requirements.txt
          pip install -r requirements-dev.txt

      - name: Lint with ruff
        run: ruff check .

      - name: Check formatting with ruff
        run: ruff format --check .

      - name: Run tests
        run: pytest tests/ -x -q --tb=short
```

Key details:
- Job name `lint-and-test` must match the name used in GitHub branch protection required status checks.
- `cache: "pip"` uses `setup-python`'s built-in pip caching keyed on requirements files.
- `ruff check .` lints for errors; `ruff format --check .` verifies formatting without modifying files.
- `pytest tests/ -x -q` runs all tests, stops on first failure (`-x`), quiet output (`-q`).
- Both `requirements.txt` (project deps) and `requirements-dev.txt` (ruff, pytest) are installed.

  <action>
Create `.github/workflows/cd.yml`:

```yaml
name: CD

on:
  push:
    branches: [main]

jobs:
  build-and-deploy:
    name: Build, Deploy, Smoke Test
    runs-on: ubuntu-latest
    permissions:
      id-token: write   # Required for OIDC token issuance
      contents: read    # Required for actions/checkout

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Configure AWS credentials (OIDC)
        uses: aws-actions/configure-aws-credentials@v4
        with:
          role-to-assume: ${{ secrets.AWS_ROLE_ARN }}
          aws-region: ${{ secrets.AWS_REGION }}

      - name: Login to Amazon ECR
        id: login-ecr
        uses: aws-actions/amazon-ecr-login@v2

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build and push to ECR
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: |
            ${{ steps.login-ecr.outputs.registry }}/${{ secrets.ECR_REPOSITORY }}:${{ github.sha }}
            ${{ steps.login-ecr.outputs.registry }}/${{ secrets.ECR_REPOSITORY }}:latest
          provenance: false
          sbom: false

      - name: Set up Terraform
        uses: hashicorp/setup-terraform@v3
        with:
          terraform_wrapper: false

      - name: Terraform init
        working-directory: ./infra
        run: terraform init

      - name: Terraform apply
        working-directory: ./infra
        run: |
          terraform apply -auto-approve \
            -var="image_tag=${{ github.sha }}"

      - name: Wait for Lambda update to propagate
        run: |
          FUNCTION_NAME=$(cd infra && terraform output -raw lambda_function_name)
          aws lambda wait function-updated \
            --function-name "$FUNCTION_NAME" \
            --region "${{ secrets.AWS_REGION }}"

      - name: Smoke test
        run: |
          API_URL=$(cd infra && terraform output -raw api_gateway_url)
          echo "Testing health endpoint: ${API_URL}/health"
          curl -fsS -o /dev/null -w "%{http_code}" "${API_URL}/health" \
            || (echo "FAIL: GET /health returned non-200" && exit 1)
          echo "Testing predict endpoint: ${API_URL}/predict"
          curl -fsS -o /dev/null -w "%{http_code}" "${API_URL}/predict" \
            || (echo "FAIL: GET /predict returned non-200" && exit 1)
          echo "Smoke tests passed."
```

Key details and rationale:
- `permissions: id-token: write` is REQUIRED for OIDC; without it, `configure-aws-credentials` fails.
- `aws-actions/configure-aws-credentials@v4`: v4 required for current OIDC support and Node 20 runtime.
- `provenance: false` + `sbom: false`: CRITICAL for Lambda compatibility. Without these, `docker/build-push-action@v4+` produces OCI format with attestation manifests that Lambda rejects with `InvalidParameterValueException`. Lambda only supports Docker Image Manifest V2 Schema 2.
- `terraform_wrapper: false`: Disables HashiCorp's output wrapper to allow clean `terraform output` parsing via shell.
- `-var="image_tag=${{ github.sha }}"`: Passes the commit SHA as the image tag Terraform variable. This ensures each merge produces a unique `image_uri` in Terraform state, forcing Lambda to pull the new image. If `latest` tag were used instead, Terraform would see no state change and skip the Lambda update.
- `aws lambda wait function-updated`: Polls until Lambda finishes updating (up to 5 minutes). Prevents smoke test hitting the old function during propagation delay.
- `curl -fsS`: `-f` = fail on HTTP errors (exits 22 for 4xx/5xx), `-s` = silent, `-S` = show errors.
- `terraform output -raw`: Extracts the raw string value without quotes. Requires `lambda_function_name` and `api_gateway_url` outputs to be defined in `infra/outputs.tf`.

Verify that `infra/outputs.tf` contains both `lambda_function_name` and `api_gateway_url` outputs. If either is missing, add them:

